The 13-Stage Process Map

The playbook is per-feature, not per-sprint: align, capture context, specify, slice, triage, design architecture, implement, review, QA, ship, debug when needed, then retro and learn.

Stage Index

00 · Prereqs

Every session starts by checking skills, repo setup, and guardrails.

01 · Align

Settle the plan before code. Open the alignment explainer.

02 · Context and ADRs

Promote settled vocabulary and durable decisions.

03 · PRD

Turn alignment into a short product requirement.

04 · Breakdown

Convert the PRD into vertical slices. Open the slice checker.

05 · Triage

Keep the issue tracker honest and decide what is worth doing.

06 · Architecture

Review module depth, data flow, and seams every few slices.

07 · Implementation

Build one slice at a time with TDD.

08 · Review

Run staff-engineer, design, devex, or security review before PR.

09 · QA

Use a real browser and add regression tests for found bugs.

10 · Ship

Open the PR, doc-close, deploy, canary, and benchmark.

11 · Debug

Diagnose before fixing. No guessing-and-hoping.

12 · Retro

Capture lessons and promote reusable patterns.
