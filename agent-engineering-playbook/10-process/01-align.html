AI Engineering Playbook · §10.01

Align Before Code

The cheapest moment to fix a misunderstanding is before code exists. Alignment is a short conversation in which the agent questions your plan — so the expensive part, the building, starts from a plan that holds up.

What is a slash command? Names like /grill-with-docs and /autoplan appear throughout these guides. They are — pre-written instruction packs installed into the agent (many come from Matt Pocock's skills collection and Garry Tan's GStack). You never type them. You write plain sentences; the agent recognises what you want and runs the right skill. The names are shown here only so you can recognise what the agent is doing.

Which Alignment Tool?

Match the shape of your request. Each card shows the sentence you actually type, what the agent does with it, and how you know alignment is finished.

The product is settled; the change is code-level

You type "I want to build [feature]."
What happens The agent the plan against the existing docs — asking the awkward questions now, while they are cheap — and records settled answers in or an .

(skill: /grill-with-docs)

Brand-new project, or you're not sure it's worth building

You type "I have an idea: [idea]. Challenge it."
What happens For a idea the agent first stress-tests whether this is the right problem at all, then sharpens the product shape, then settles the technical questions.
Done when You can say in one sentence what you are building and why.

(skills: /office-hours → /plan-ceo-review → /grill-with-docs)

Your screen/UI is changing

You type "Before we code, make a mockup."
What happens The agent builds a fake HTML version of the screen — a — so you react to a picture, not a description.
Done when You've said "yes, build that" to a mockup.

(skill: /plan-design-review)

You're building something for developers

You type "Review this plan as a developer would experience it."
What happens The agent checks the API, command-line tool, or docs for : how quickly a stranger gets their first success, and how clear the errors are when they get something wrong.
Done when The first-use journey is written down and survives the review.

(skill: /plan-devex-review)

You want every angle covered

You type "Run a full review of this plan."
What happens The agent applies the business, design, and engineering lenses in sequence, without you having to choose the order.
Done when All three reviews pass, or their concerns are written down.

(skill: /autoplan)

What A Grilling Actually Looks Like

Here is the booking-form feature being grilled. Notice that the agent is not being difficult — every question is a that will not happen later because it was asked now.

YouI want people to be able to book a session online.
AgentWhat should happen if two people pick the same slot at the same time?
YouFirst one gets it; the second should see it's gone.
AgentCan people cancel? If so, until when?
YouYes — up to 24 hours before the session.
AgentNoted. I'm recording slot and booking window in CONTEXT.md so we use the same words everywhere. Does a booking need anything beyond a name and a time?
YouAn email confirmation.
AgentAdded. I have enough to write the one-page spec.

Exit Criteria

Alignment is finished when all of these are true:

  • Every open question has an answer or a written "not now".
  • The project's words are written down in , so human and agent use the same names for the same things — no .

  • You have said: go ahead.
