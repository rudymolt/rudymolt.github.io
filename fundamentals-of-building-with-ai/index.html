The question

Have you ever asked AI to do something —
not just answer something?

Most of us use AI as a clever search box. Today is about the leap from using it to building with it.

1 · Hook
The one idea to remember
You're no longer the person filling in the spreadsheet.
You're the person who briefs and supervises the thing that fills it in.

You give it a job — "build me a dashboard from this season's results" — it works, you check it.

2 · Why it's different

Predictable vs probabilistic

Deterministic

Traditional software

Predictable — same input, same output, every time. Click "bold", it bolds. Rigid, but never surprises you.

Probabilistic

AI

Works from likelihoods, not fixed rules. Handles mess and plain English — but can be inconsistent, and occasionally wrong.

Vending machine (deterministic): button B4 always drops the same crisps. Assistant coach (probabilistic): "sort tomorrow's warm-up, the pitch is waterlogged" — they just work it out. AI is the coach.

What you get out depends on the quality of what you put in.

3 · Mental model

The model — the brain

The raw intelligence. It can think, write and reason — but on its own it has no hands: it can't open your files or save a document.

In the analogy: the athlete's natural talent. Brilliant, but does nothing sitting on the sofa.

Real examples (June 2026): the leading "frontier" models —

GPT-5.5 · OpenAI Claude Opus 4.8 · Anthropic Gemini 3.1 Pro · Google Grok 4.3 · xAI

Version numbers are just like iPhone 15 → 16. "Frontier" = one of the most capable available right now.

3 · Mental model

The harness — the body & kit

Everything built around the brain that lets it actually do work: access to your files, tools it can use, a safe workspace, memory, and the loop to act → check → act again.

Why it matters: two people, same model, wildly different value — one types into a chat box, the other has set it up to read data, build files and check itself. Same brain; better body.

3 · Mental model
Common point of confusion

Harness vs agent: same thing, two views

They're not two products you choose between. The harness is the machinery; the agent is that machinery running with a model, doing your job.

  • Codex and Hermes sit in the same box — both harnesses that run as agents.
  • Codex the software = harness · Codex mid-task = agent.
  • The difference between them is design, not category (Codex: OpenAI-tied, coding-focused · Hermes: model-agnostic, self-hosted, persistent memory).
Pick a harness (Codex, Claude Code, Hermes…), plug in a model (GPT-5.5, Claude…) — the running combination is the agent.

4 · Three habits
Habit 1

Keep information where AI can read it

Put information in a database or markdown / plain-text files — not trapped in scanned PDFs, slide decks, email threads, or people's heads.

Markdown = plain text with simple structure marks (# for a heading). Machine-readable = laid out so software can parse it — not a photo of a whiteboard.

Squad A keeps results in a clean shared sheet → AI analyses the whole season instantly. Squad B keeps them in a notebook and a WhatsApp thread → AI can do almost nothing until someone types it up.

Tidying your information is the build work — not preparation for it.

4 · Three habits
Habit 2

Produce outputs as HTML

When you want something to look at or share — a report, a dashboard — ask for it as HTML.

HTML = the language of web pages. Opens in any browser on any device, needs nothing installed, and can be interactive: charts, filters, clickable tables.

Instead of a static PDF league table emailed round → an HTML dashboard where coaches filter by player, sort by metric, watch a chart update live — opened from one link on a phone at the side of the pitch.

"Which version of Office?" becomes "just open this in your browser."

4 · Three habits
Habit 3

Stay in the loop

Treat AI work as a loop — brief → review → correct → improve — not "submit and forget".

Iterative = improved in repeated passes. Confidently wrong ("hallucination") = stating something false in a convincing tone — a side-effect of being probabilistic. It's why a human checks the figures.

Example: a match report's first draft mislabels an assist. You spot it — "the second goal was Jones, not Smith; re-check the others" — corrected version in seconds. Trust, but verify.

4 · Three habits
Section recap

Three habits, one theme

Feed it clean information · ask for shareable outputs · keep a human in the loop.

Get these right and everything in the later sessions becomes easy.

Section 5 · going deeper

Setting up your harness

How you set up Claude or Codex matters more than which one you pick.

Same model, same harness, two people — completely different results. The difference is setup.

5 · Setting up your harness

You're not buying a tool —
you're inducting a new staff member

No induction

A brilliant new analyst with no induction keeps asking where things are, uses the wrong template, and guesses your standards.

Properly inducted

Same person — handed the handbook, shown where the data lives, told how you like things — is useful on day one.

Setting up your harness is that induction.
5 · Setting up your harness
The principle

Context engineering over prompt engineering

A clever prompt

One well-worded instruction. Helps once, then it's gone.

Good context

The standing brief, data access, house style and boundaries you build around the agent. Pays off on every task, forever.

The real skill isn't one clever request — it's the setup the agent works inside. Configuration compounds.

5 · Setting up your harness
The high-leverage three

Brief it · connect it · teach it

Context file

The induction handbook

Who you are, where data lives, what columns mean, your conventions — read automatically every time.

Sport: "Results live in /data/results as CSV; PB = personal best; times in seconds to 2 dp; British English, metric."

Connectors · MCP

Hands into your systems

A secure link to your database, calendar or files. MCP is the universal adapter that makes it possible.

Sport: connect results DB + calendar → "build a one-pager on next week's fixtures and each opponent's form."

Skills

Reusable playbooks

Your standard way of doing a recurring task, written once so the result is consistent every time.

Sport: a "match report" skill — same sections every time: score, key moments, ratings, next up.

5 · Setting up your harness
The other levers

Standards · boundaries · order

House style

Standards, baked in

Set tone, format, units and language once; they apply everywhere, so you stop correcting the same things.

Live example: this project is set to British English, metric, markdown — which is why these notes came back this way.

Permissions · sandbox

What it can touch

Decide which folders it can change and whether it must ask before deleting. A sandbox is a safe walled-off workspace.

Sport: "Read the results folder, write to reports, never edit raw data, always ask before deleting."

Folder structure

Findable information

A tidy, predictable layout makes a capable agent; chaos wastes its intelligence on hunting.

Sport: /athletes · /results · /reports · /templates beats 400 files called final_v2_REAL.xlsx.

5 · Setting up your harness

Key points to take away

  • Setup beats selection. How well you configure it matters more than Claude vs Codex.
  • Configuration is reusable; prompts are disposable. Setup pays off on every future task.
  • Start small and grow it. A short context file and a tidy folder first; add connectors and skills when a task demands.
  • It's a living setup. Whenever you correct the same thing twice, put it in the context file, memory or a skill.
  • This is the real skill. Learning to configure an agent well is the transferable craft.
